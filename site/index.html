group_08 · pre-publication amendments · one claim struck pre-publication, preserved per audit contract
claim_08 verifying

An earlier draft made a claim about prompt caching that the audit struck.

The audit pass caught one claim in manifest.md draft 03, line 47. The original is preserved here, struck-through; the corrected version is the one that appears elsewhere in this page. The page is honest about its own edit history.

draft_03 · line 47 · 2026-05-12T19:14:08Z struck

"Prompt caching keeps every verification call cheap. Cache hits apply to all calls beyond the first against any workspace."

audit · 2026-05-13T08:47:38Z · trace_01KR4PV6Z3WB8YQ7 reason

Contradicted by orc's own deployment-log trace. Anthropic prompt caching requires a minimum cached prefix (≈3K tokens for sonnet, ≈2K for haiku). Workspaces with small corpora produce smaller prefixes and may never trigger cache writes. The blanket claim misled the reader on cost behavior.

draft_04 · line 47 · 2026-05-13T08:51:02Z corrected

"Prompt caching is effective for workspaces whose retrieval produces >3K tokens of cached prefix per call (Sonnet) or >2K (Haiku). Smaller corpora may not trigger cache hits; for those, each call is priced as full input. The trace records cache_creation_input_tokens and cache_read_input_tokens so cost behavior is transparent."

trace_01KR4PV6Z3WB8YQ7 deployment-log · cache-behavior · §3
cache_create=2412 · cache_read=0 · across 4 successive calls against an identical workspace prefix of 2412 tokens (below sonnet caching threshold). Final verdict: cache writes occurred, cache reads did not. Cost was not reduced.
