The discipline

Four invariants the runtime enforces by construction.

Most "AI safety" tooling layers checks on top of generation. Orc inverts that — the architecture makes failure modes structurally impossible, not just unlikely.

01 — Citations

Cannot fabricate sources.

Every chunk ID Orc returns is validated against the actual retrieval set. Hallucinated citations are dropped before the verdict reaches you. The model is structurally prevented from inventing a source.

02 — Architecture

Stateless skills, durable context.

Skills are pure functions with explicit input/output contracts. No agent identities, no persistent personas, no emergent coordination. Persistence lives in the workspace — not in the agents — so behavior is auditable and reproducible.

03 — Replay

Every run is reconstructable.

Every CLI or MCP call writes a trace: retrieval set, every LLM call's tokens and cache hits, the structured output. orc replay <run_id> re-executes the exact decision against the same corpus snapshot. Months later. Same answer. Same evidence.

04 — Approval

Analysis ≠ action.

Anything that would mutate the outside world lands in the approval queue first. Humans accept or reject with a note. Write paths run as separate processes with separate tokens. The blast radius of a compromised agent is, by design, zero.

How it works

A workspace. A corpus. A claim. Receipts.

The whole loop in four commands. Storage is local SQLite under ~/.orc/. Bring your own Anthropic key, OpenRouter key, or local model. The CLI is the same surface other agents see through MCP.

~/orc — the four-step loop 
# 1. A workspace is a trust boundary. One per matter, project, or topic.
$ orc workspace create research
Created workspace research

# 2. Ingest evidence: files, directories, or URLs. Markdown, txt, json.
$ orc ingest ./notes -w research
$ orc ingest https://www.anthropic.com/engineering/multi-agent-research-system -w research
Ingested 14 evidence items into research

# 3. Verify any claim. Or pipe a whole draft through.
$ orc verify --file draft.md -w research --yes
7 claims extracted from draft.md. Verifying...

  1. supported      (0.92)  Anthropic released the Skills API in October 2025
  2. partial        (0.71)  The multi-agent eval delta was 90.2%
  3. supported      (0.88)  Prompt caching has a 5-minute ephemeral TTL by default
  4. contradicted   (0.95)  Multi-agent setups always outperform single-agent
  5. supported      (0.86)  Skills are versioned, auditable capabilities
  6. not_found      (0.90)  Anthropic was acquired by Microsoft in 2026
  7. supported      (0.94)  Tool use is more reliable than JSON-mode for structured output

# 4. Inspect, replay, or approve any decision later.
$ orc trace show 01KR1NH...
$ orc replay 01KR1NH... --live

Common questions

The honest answers.

Why not just use Claude Code or ChatGPT directly?
For one-off questions, you should. Direct LLM chat is faster and free. Orc is for the cases where you need: (a) a permanent paper trail you can audit later, (b) a stable trust boundary so the answer comes only from sources you've vetted, (c) bulk verification of a draft or a critique, or (d) integration into a programmatic workflow with structured JSON output. Think of Claude as the smart friend who's read a lot. Orc is your librarian who only answers from the books you put on the shelf.
Which models does Orc use?
Anthropic Claude models by default — directly via the Anthropic API or routed through OpenRouter, whichever you have credentials for. Local model support (vLLM, Ollama) is on the Enterprise tier for deployments where data can't leave the network.
Is the CLI open source?
Yes. The CLI, the verification skills, and the trace/replay layer are MIT-licensed. The hosted dashboard and enterprise integrations are commercial.
What's the alternative to me writing fact-checking myself?
That's the point. Orc is the runtime so you don't have to glue together retrieval, citation validation, prompt-cache discipline, traces, and approval queues for every workflow that needs verification. Bring evidence, get receipts.
What does Orc not do?
It doesn't generate freeform content unconstrained by evidence. It doesn't take autonomous external actions — only proposes them for human approval. It doesn't search the open web on its own; it only knows what you ingest. These are deliberate constraints, not gaps.
